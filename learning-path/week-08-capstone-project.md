# Week 8: Capstone Project

## Goal

Use the existing task tracker as your capstone and extend it until you can explain every part of it.

## Required Features

- add a task
- list tasks
- mark a task complete
- delete a task
- save and load JSON
- validate input cleanly
- handle missing and corrupt files safely
- keep tests passing

## Stretch Features

- due dates
- priorities
- filtering active vs completed tasks
- better terminal output

## Suggested Challenge Order

1. Add `--completed-only` and `--active-only`
2. Add due-date validation
3. Sort tasks by priority
4. Show overdue tasks differently
5. Add more tests for new behavior

## Final Review Check

- Can you trace a command from CLI input to saved JSON?
- Can you explain every class and function in the project?
- Can you write one new feature without breaking tests?
